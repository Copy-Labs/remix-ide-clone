Explanation of the Implementation

The Git branch persistence has been moved from localStorage to IndexedDB for improved reliability and storage capacity. This change provides several benefits:

  • More Storage Space: IndexedDB provides significantly more storage space than localStorage (typically 50MB+ vs 5MB).
  • Better Performance: IndexedDB is designed for storing larger amounts of data efficiently.
  • Asynchronous API: IndexedDB operations don't block the main thread, improving application responsiveness.
  • Structured Data: IndexedDB allows storing complex data structures more naturally.

The implementation uses a custom storage adapter for Zustand that interfaces with the application's existing databaseService, which manages IndexedDB operations.
